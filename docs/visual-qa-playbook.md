# 肝视界 Demo 视觉 QA 验收手册

这份手册用于完成浏览器桌面 / 移动端 QA 与 Three.js 画布可视检查。当前 Codex Browser 访问 `http://127.0.0.1:5173` 被企业网络策略拦截，且策略禁止通过其它浏览器面或底层命令绕过；因此本手册作为后续有浏览器权限时的标准验收路径。

## 准备

```bash
pnpm install
pnpm qa:self-check
pnpm dev
```

打开：

```text
http://127.0.0.1:5173/
```

验收视口：

- 桌面：`1440 x 960`
- 移动：`390 x 844`

每个视口都需要打开 DevTools Console，确认没有红色运行时错误。

通用验收标准：

- 页面不能出现横向滚动条。
- 导航、按钮、筛选控件、卡片标题和关键指标不能重叠或截断。
- 图片不能破图，卡片与列表在移动端应保持清晰层级。
- 主要操作按钮点击后应有状态反馈，且状态枢纽能同步最近动作。
- 本地刷新后，病例选择、试点材料、报告意见、影像证据等演示状态应保留。

## 3D 画布验收

在 `/imaging` 执行：

1. 确认 `Three.js` 画布区域非空白，肝脏、肿瘤和血管结构可见。
2. 切换模型预设：`术前规划`、`血管优先`、`病灶定位`，确认可视重点变化。
3. 调整透明度，确认肝脏透明度变化可见。
4. 用鼠标拖拽旋转模型，用滚轮缩放，确认交互无卡死。
5. 点击截图 / 导出类按钮，确认影像证据条和状态枢纽记录动作。

建议证据：

- `desktop-imaging-canvas.png`
- `mobile-imaging-canvas.png`
- `desktop-imaging-after-preset.png`

3D 画布通过标准：

- 画布尺寸在桌面和移动端均大于 `280 x 220`。
- 模型主体非空白，至少能同时辨认肝脏轮廓和一个病灶或血管结构。
- 交互后画面仍然稳定渲染，没有 WebGL context lost、白屏或控制台错误。

## 端到端演示路径

完成以下三条路径，并记录每一步截图或短录屏：

1. 驾驶舱闭环：`/dashboard` 选择 P1 病例，进入 `/imaging`，启动 AI 处理，完成影像证据，生成报告。
2. 报告闭环：`/reports` 选择同一病例，生成复核意见，导出 MDT 材料，确认导出历史更新。
3. 试点闭环：`/pilot` 配置医院类型、部署路径、病例包和材料清单，提交申请，生成交付简报。
